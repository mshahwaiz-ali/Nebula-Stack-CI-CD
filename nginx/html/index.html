NebulaStack

Mini production-style stack: Nginx web, Flask API, PostgreSQL DB, Docker Compose orchestration, GitHub Actions CI/CD aur AWS EC2 deploy – sab ek place pe.

Dockerized Stack Reverse Proxy REST API Postgres CI/CD to EC2
Project Snapshot

NebulaStack ek simple lekin realistic setup hai: static site Nginx se serve hoti hai, Flask API background me run hoti hai, PostgreSQL messages store karta hai aur sab containers Docker Compose se ek command me start hote hain. GitHub Actions push pe EC2 pe latest version deploy karta hai.

Nginx Web Flask API PostgreSQL 16 Docker Compose GitHub Actions AWS EC2 + SSH
Fast Overview
  • ● Stack start: docker compose up -d --build
  • ● Public entry: http://EC2_IP:8080
  • ● API check: /api/health, DB check: /api/db-test
  • ● Messages: /api/messages (list, create, delete)
  • ● CI/CD: push to main ⇒ build + test + deploy on EC2.

Project Breakdown – Components & Features

Frontend

Nginx Web Container

Static HTML/CSS overview page serve karta hai aur port 8080 expose karta hai. Ye hi entry point user ke liye.

  • Base image: nginx:alpine, custom config + site copy.
  • Static files: /usr/share/nginx/html se serve.
  • Check: browser me http://EC2_IP:8080 load hona chahiye.
Backend API

Flask Application

REST API jo health, DB test aur messages endpoints expose karti hai. Nginx ke through /api prefix se access.

  • GET /api/health → { status: "ok", env: ... }
  • GET /api/messages → saare messages list.
  • POST /api/messages → { "content": "..." } save.
Database

PostgreSQL Messages DB

Messages persistent storage PostgreSQL 16 container handle karta hai, API directly is DB se connect hoti hai.

  • DB: nebulastack / nebula / secretpass.
  • Table: messages(id, content, created_at).
  • Check: GET /api/db-test simple SELECT 1.
Orchestration

Docker & Docker Compose

Sab containers ek docker-compose.yml file se manage hote hain – same network, defined ports, restart rules.

  • Services: web, api, postgres.
  • Command: docker compose up -d --build.
  • Internal DNS: service naam hostnames ki tarah kaam karte hain.
Routing

Reverse Proxy Config

Nginx static site + API dono handle karta hai, aur basic security headers ke options allow karta hai.

  • location / → static HTML response.
  • location /api/ → proxy to api:5000.
  • Idea: headers jese X-Frame-Options, X-Content-Type-Options.
CI/CD

GitHub Actions Workflow

Repo pe push hone pe pipeline run hoti hai: code check, image build aur EC2 pe deploy.

  • Job 1: tests + Docker image build/push.
  • Job 2: needs: api-ci ke baad SSH se EC2 login.
  • Deploy: git pull + docker compose up -d --build.
Cloud

AWS EC2 + SSH Secrets

Live stack Ubuntu-based EC2 instance pe run hota hai, jahan Docker + Compose install hai aur repo clone hai.

  • Elastic IP use, taake public IP stable rahe.
  • Secrets: EC2_SSH_KEY, EC2_HOST, EC2_USER.
  • Access: ssh ubuntu@EC2_HOST same key se.
Checks

Health & Quick Tests

Basic curl commands se stack quickly verify kiya ja sakta hai – API, messages aur logs.

  • API: curl http://localhost:8080/api/health
  • Messages read: curl http://localhost:8080/api/messages
  • Messages create (short):
    curl -X POST /api/messages -d '{"content":"test"}'
